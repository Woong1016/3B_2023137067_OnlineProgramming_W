Firebase 유저 데이터 관리 시스템 과제 안내
과제명
Firebase를 활용한 게임 유저 데이터 관리 시스템 구현
과제 목표
이번 과제는 Unity와 Firebase Realtime Database를 연동하여 게임에서 자주 사용하는 유저 데이터 관리 기능을 구현하는 과제입니다.
수업에서 배운 내용을 바탕으로 다음 기능을 직접 완성해야 합니다.

회원가입
로그인
코인 불러오기
아이템 구매
인벤토리 표시
아이템 사용
Firebase 데이터 저장 및 갱신
이번 과제의 핵심은 단순히 코드를 복사하는 것이 아니라, 로그인한 유저의 데이터를 Firebase에서 불러오고, 수정하고, 다시 저장하는 흐름을 이해하는 것입니다.
제공 코드
기본 코드 파일이 제공됩니다.
제공되는 스크립트는 다음과 같습니다.

UserData.cs
UserRegister.cs
UserLogin.cs
ShopManager.cs
InventoryManager.cs
제공 코드는 수업에서 배운 기본 구조를 포함하고 있습니다.
다만, 제공 코드에 있는 아이템 이름을 그대로 사용하면 안 됩니다.
중요 조건
제공 코드에는 예시로 다음 아이템이 들어 있습니다.

Potion
Bomb
Ticket
하지만 과제 제출 시에는 위 아이템을 그대로 사용하면 안 됩니다.
반드시 본인 게임에 맞는 새로운 아이템 3개 이상으로 변경해야 합니다.
예시:

EnergyDrink
Shield
MagicStone
GoldKey
RepairKit
FireScroll
IceCrystal
GachaCoin
아이템 이름, 가격, 사용 메시지는 본인이 직접 정해야 합니다.
필수 구현 기능
1. 회원가입 기능
닉네임을 입력하여 회원가입할 수 있어야 합니다.
필수 조건:

닉네임을 입력하지 않으면 회원가입이 되지 않아야 합니다.
이미 존재하는 닉네임이면 회원가입이 되지 않아야 합니다.
회원가입 성공 시 Firebase에 유저 데이터가 저장되어야 합니다.
회원가입 성공 시 UserKey와 UserNickName이 PlayerPrefs에 저장되어야 합니다.
회원가입 시 기본 데이터 예시:

{
  "NickName": "Player01",
  "Coin": 500,
  "Score": 0,
  "UnitList": "{\"Unit1\":true,\"Unit2\":false,\"Unit3\":false}",
  "Inventory": "{\"EnergyDrink\":0,\"Shield\":0,\"MagicStone\":0}"
}
2. 로그인 기능
이미 회원가입한 닉네임으로 로그인할 수 있어야 합니다.
필수 조건:

닉네임을 입력하지 않으면 로그인되지 않아야 합니다.
존재하지 않는 닉네임이면 로그인되지 않아야 합니다.
로그인 성공 시 UserKey와 UserNickName이 PlayerPrefs에 저장되어야 합니다.
로그인 성공 후 상점 또는 인벤토리 화면으로 이동할 수 있어야 합니다.
3. 코인 불러오기 기능
로그인한 유저의 코인을 Firebase에서 불러와 UI에 표시해야 합니다.
필수 조건:

PlayerPrefs에 저장된 UserKey를 사용해야 합니다.
Firebase의 UserInfo/UserKey/Coin 값을 불러와야 합니다.
불러온 코인을 Text UI에 표시해야 합니다.
예시:

Coin : 500
4. 아이템 구매 기능
상점에서 아이템을 구매할 수 있어야 합니다.
필수 조건:

아이템은 본인이 새롭게 정한 아이템 3개 이상이어야 합니다.
각 아이템은 서로 다른 가격을 가져야 합니다.
구매 버튼을 누르면 현재 코인을 확인해야 합니다.
코인이 부족하면 구매가 되지 않아야 합니다.
코인이 충분하면 코인이 차감되어야 합니다.
구매한 아이템의 개수가 1 증가해야 합니다.
변경된 Coin과 Inventory가 Firebase에 저장되어야 합니다.
구매 결과 메시지가 UI에 표시되어야 합니다.
예시:

EnergyDrink : 100 Coin
Shield : 200 Coin
MagicStone : 300 Coin
구매 전:

Coin : 500
Inventory : {"EnergyDrink":0,"Shield":0,"MagicStone":0}
EnergyDrink 구매 후:

Coin : 400
Inventory : {"EnergyDrink":1,"Shield":0,"MagicStone":0}
5. 인벤토리 표시 기능
Firebase에 저장된 인벤토리 정보를 불러와 화면에 표시해야 합니다.
필수 조건:

Firebase에서 Inventory 값을 불러와야 합니다.
Newtonsoft.Json을 사용하여 JSON 문자열을 Dictionary로 변환해야 합니다.
본인이 만든 아이템 3개 이상의 개수를 UI에 표시해야 합니다.
예시:

EnergyDrink : 2
Shield : 1
MagicStone : 0
6. 아이템 사용 기능
인벤토리에 있는 아이템을 사용할 수 있어야 합니다.
필수 조건:

아이템 개수가 0이면 사용할 수 없어야 합니다.
아이템을 사용하면 해당 아이템 개수가 1 감소해야 합니다.
변경된 Inventory가 Firebase에 다시 저장되어야 합니다.
사용 결과 메시지가 UI에 표시되어야 합니다.
아이템별 사용 메시지를 다르게 출력해야 합니다.
예시:

EnergyDrink 사용 완료
Shield 사용 완료
MagicStone 사용 완료

Firebase 데이터 구조
Firebase Realtime Database에는 다음과 같은 구조로 저장되어야 합니다.

UserInfo
 └── 유저고유키
      ├── NickName
      ├── Coin
      ├── Score
      ├── UnitList
      └── Inventory
예시:

{
  "UserInfo": {
    "-NxABC123": {
      "NickName": "Player01",
      "Coin": 400,
      "Score": 0,
      "UnitList": "{\"Unit1\":true,\"Unit2\":false,\"Unit3\":false}",
      "Inventory": "{\"EnergyDrink\":1,\"Shield\":0,\"MagicStone\":0}"
    }
  }
}

과제 핵심 정리
이번 과제에서 가장 중요한 것은 화려한 UI가 아닙니다.
중요한 것은 다음 흐름을 이해하는 것입니다.

로그인
→ UserKey 저장
→ UserKey로 내 데이터 찾기
→ Coin / Inventory 불러오기
→ 데이터 수정
→ Firebase에 다시 저장
온라인 게임에서는 유저의 데이터가 계속 저장되고 수정됩니다.
이번 과제를 통해 Firebase를 이용한 기본적인 유저 데이터 저장 구조를 이해하기 바랍니다.
